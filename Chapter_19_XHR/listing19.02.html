Loading just data

Adventures in The Crypt can involve exploring ancient tombs, sprawling spaceships and mysterious forests. Each adventure may cover dozens of locations with little chance of completion in a single sitting. Rather than loading the whole map at the start of a game, it might make more sense to load places only as they are visited. The JSON data for each place will be stored in a separate file and we’ll use XMLHttpRequest to load the files when needed.

As a map is explored, a player will visit the same locations many times. We don’t want to keep loading the same data across the Internet. Whenever a new place is created from loaded data, the place will be stored in a cache, using the place’s id as the key. Further requests for a place can be given the stored place directly.
